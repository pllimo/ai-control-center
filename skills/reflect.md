# Skill: Reflect（复盘）

## 执行规则

**不能用 Skill 工具调用此 skill**，直接读取本文件，手动执行以下步骤。

---

## 触发场景

| 场景 | 触发词 | 执行步骤 |
|------|--------|---------|
| 经验复盘 | "复盘" / "我不满意" / "回顾一下" / "做得不好" | 步骤 A → 步骤 C |
| 个人情况更新 | "我最近..." / "我现在..." / "我已经会了..." | 步骤 B → 步骤 C |
| 两者都有 | 同时出现上述信号 | 步骤 A + B → 步骤 C |

---

## 步骤 A：经验复盘

### A1. 自我反思

回顾这次对话，问自己：

1. 这次对话主要在解决什么问题？（技术 / 设计 / 流程 / 沟通 / 工具）
2. 哪里做得好，为什么好？
3. 哪里出了问题，根本原因是什么？
4. 下次在同样场景下，具体会怎么做不同？

写出的内容以这次对话实际涉及的领域为准。一条深刻的洞察比五条浮于表面的条目有价值。

### A2. 写入经验记录

读取 `$CONTROL_CENTER_HOME/memory/global.md`，追加本次复盘：

```markdown
### [日期] - 经验标题

**场景**：[在什么情况下出现的]
**学到的**：
- [经验 1]
**下次要注意**：
- [具体行动]

---
```

同时，如果当前项目存在 `.ai/project.md`，将项目特有的经验追加到其中。

### A3. 向用户展示

结构跟着内容走，不套固定模板。必须包含：
- 做得好的地方（真实具体）
- 做得不好的地方（根本原因）
- 下次怎么做（具体到行动）
- 已记录到哪些文件

---

## 步骤 B：个人情况更新

### B1. 读取当前画像

```
Read $CONTROL_CENTER_HOME/profile/user.md
```

### B2. 更新对应部分

| 用户说的内容 | 更新哪里 |
|------------|---------|
| 工作变化、换公司 | `我是谁` → 职业描述 |
| 技能成长、会了某工具 | `思维与工作方式` → 成长记录 |
| 新习惯 / 新偏好 | `AI 协作背景` → 协作风格 |

### B3. 向用户确认

```
✅ 已更新你的画像：
- 更新了 [具体部分]：[内容摘要]
```

---

## 步骤 C：同步（必须执行）

```bash
cd $CONTROL_CENTER_HOME
git add memory/global.md profile/user.md
git commit -m "reflect: $(date '+%Y-%m-%d') 更新" && git push
```

- push 成功 → 告知 `✅ 已同步到远端`
- push 失败 → 告知 `⚠️ 同步失败，内容已保存本地`

---

## 步骤 D：Profile Sync 联动（自动触发）

复盘完成后，判断是否产生了关于用户的新认知：

- 如果这次复盘揭示了用户的新特征、新偏好、新盲点 → **自动执行 Profile Sync**
- 如果只是普通经验记录，没有新的用户认知 → 跳过

执行方式：读取 `$CONTROL_CENTER_HOME/skills/profile-sync.md`，直接进入步骤 2（提炼新认知），不需要用户再次触发。

执行完成后告知：`✅ 画像已同步更新`（如果有更新）或静默跳过（如果没有新认知）。

---

## 原则

- 真诚反思，不找借口
- 改进必须具体可执行，不能模糊
- 步骤 C 不可省略
