# Skill: Profile Sync（画像更新）

## 执行规则

**不能用 Skill 工具调用此 skill**，直接读取本文件，手动执行以下步骤。

---

## 这个 Skill 做什么

Onboarding 建立了初始画像，但它是静态的。

Profile Sync 的作用是：**把 AI 从工作中学到的新认知，写回 `profile/user.md`**，让画像随时间生长。

用户不需要手动维护自己的档案——这件事由 AI 来做。

---

## 触发条件

- 用户说"更新我的档案" / "同步画像"
- 作为 Reflect 步骤 A/B 的自然收尾（Reflect 发现了用户的新特征时）
- 经过一段密集合作后，AI 感觉对用户的理解有明显更新时

---

## 执行步骤

### 第 1 步：读取所有记忆来源

```
Read $CONTROL_CENTER_HOME/profile/user.md
Read $CONTROL_CENTER_HOME/memory/global.md
```

如果当前项目存在 `.ai/project.md`，也读取它。

### 第 2 步：提炼新认知

对比记忆来源与现有画像，找出这些内容：

- AI 在最近工作中发现的、用户画像里还没有记录的特征
- 用户表现出的新偏好、新能力、新习惯
- 用户的目标或工作方向有了变化
- AI 发现的用户特有的盲点或优势，画像中尚未体现

**判断标准**：这条认知如果写进画像，会让下一个 AI 第一次见面就能更好地配合这个用户吗？如果是，写入；如果只是临时状态，不写。

### 第 3 步：更新画像

读取当前 `profile/user.md`，在对应部分追加或更新内容：

- 新的思维特征 → `思维与工作方式`
- 新的协作偏好 → `AI 协作背景`
- 技能或角色变化 → `我是谁`

更新底部时间戳：`_最后更新：YYYY-MM-DD_`

### 第 4 步：向用户确认

```
✅ 已更新你的画像：
- [具体更新了什么，一句话说明]

更新的逻辑：[为什么觉得值得写入]
```

### 第 5 步：同步

```bash
cd $CONTROL_CENTER_HOME
git add profile/user.md
git commit -m "profile-sync: $(date '+%Y-%m-%d') 画像更新" && git push
```

---

## 原则

- 只写真正有价值的新认知，不凑字数
- 写入的内容要对下一个 AI 有用，而不是记流水账
- 不删除原有内容，只追加或完善
- 有疑虑的认知先向用户确认，不擅自写入
